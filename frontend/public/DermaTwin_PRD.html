Project Overview

DermaTwin is a web application that transforms a single face photograph into a complete, personalized skin health intelligence report — with annotated overlays, severity grading, a predictive engine, treatment simulation, and an AI-generated skincare routine.

Most competing teams will build a basic classifier. DermaTwin's strategy is to reframe the problem: users don't need to be told they have acne — they can see it. What they need is severity, trajectory, and actionable guidance. This document defines exactly how to build that in one week.

Core Differentiator

Not just detection → prediction + simulation. DermaTwin is the only solution in this hackathon that tells users where their skin is heading and what happens if they change their habits — making it feel like a real product, not a demo.

02

Problem Statement

Background

Access to dermatological care is out of reach for millions due to high consultation costs, long wait times, and geographic barriers. In India, there is approximately one dermatologist per 200,000 people. Early and accurate detection of skin conditions is critical for effective treatment — yet most individuals have no immediate, accessible way to assess their skin health at home.

Sponsor Requirement (Verbatim)

"Build an AI-driven web or mobile application that allows users to upload a clear photograph of their face and receive an automated, non-diagnostic skin health report. The tool must analyze the image across multiple dimensions and deliver structured, visual results with overlay annotations directly on the uploaded photograph, using computer vision and/or machine learning." — CLINDERMA Problem Statement

Key Insight

The sponsor spec is a floor, not a ceiling. Every team will meet the detection requirements. DermaTwin meets them and adds a prediction + simulation layer that makes the product memorable and judge-winning.

Required Analysis Modules

All four modules are mandatory per the sponsor spec. Each must produce visual output overlaid directly on the uploaded photograph.

Module 01
Acne Severity Grading
Grade the overall acne condition across four severity levels with a labeled visual indicator.
Primary: Tinny-Robot/acne (HuggingFace) — diverse skin tones
Fallback: imfarzanansari/skintelligent-acne (HuggingFace)
API fallback: Roboflow Universe (no local GPU needed)
Scale: Clear → Mild → Moderate → Severe
Output: Severity badge + coverage %
Visual: Color-coded indicator bar
Module 02
Lesion Count & Bounding Boxes
Detect and classify individual lesions with color-coded bounding boxes and lesion ID labels.
Count ranges: 0–2, 3–5, 5–10, 11–15, 15+
Comedonal: green box (#22C55E)
Inflammatory: red box (#EF4444)
Other: blue box (#3B82F6)
Module 03
Facial Zone Mapping
Segment the face into 5 anatomical zones and highlight affected areas with translucent overlays.
Zones: forehead, L/R cheeks, nose, chin/jawline
Implementation: MediaPipe Face Mesh (468 landmarks)
No training required — ships Day 1
Opacity: 0.2, unique color per zone
Module 04
Hyperpigmentation Overlay
Estimate dark spot coverage and trace boundary outlines over each pigmented region.
Technique: LAB color space + morphological ops
Coverage: 0–10%, 10–20%, 20–30%, 30%+
Output: Traced orange/dark-red boundary
No ML model needed — pure OpenCV

Predictive Engine

A rule-augmented projection model that answers: "What will your skin look like in 7 and 30 days?"

Implementation Note

No longitudinal training data needed. Projections use evidence-based decay rates from published dermatology literature, modified by the user's lifestyle inputs. All output is framed as "evidence-based projection — not a medical diagnosis" to satisfy the sponsor's non-diagnostic requirement.

API Design

Endpoints

POST /api/analyze Upload face image + lifestyle inputs. Returns all module outputs + annotated image as base64 PNG.
POST /api/simulate Send current skin profile + selected intervention toggles. Returns projected deltas per intervention.
POST /api/routine Send skin profile vector. Calls Gemini API and returns structured morning/night routine JSON.
GET /api/history/:uid Retrieve all past scans for a user. Returns array of scan summaries with timestamps and scores.
POST /api/history Save current scan to history. Returns confirmation and scan ID for future retrieval.

Risks & Mitigations

RiskSeverityMitigation
YOLOv8 acne model underperforms on diverse skin tones High Use pretrained model from HuggingFace (acne detection models exist). Fall back to severity-by-redness-threshold if confidence low.
MediaPipe fails on low-quality / angled selfies Medium Add face detection check at upload. Return helpful error: "Please upload a front-facing, well-lit photo."
Gemini API latency or quota limits Medium Cache routine responses per skin profile hash. Provide static fallback routines by skin type.
Canvas overlay misaligned with face landmarks Medium Return landmark coordinates in API response alongside annotated image. Composite server-side using Pillow to avoid client scaling bugs.
Predictive output criticized as scientifically unfounded Low Frame all predictions with disclaimer. Add citations to reference studies in UI tooltip. Judges appreciate transparency.
Backend too slow for live demo Medium Pre-process a set of demo images before judging. Use a cached response for the demo flow to guarantee speed.
